커밋 완료 후 `local/progress-issue-N.md` 체크포인트 파일을 업데이트한다.

## Steps

1. **현황 파악** (병렬로 실행)
   - `git branch --show-current` — 이슈 번호 추출
   - `git log --oneline -5` — 최근 커밋 해시 + 메시지 확인
   - `local/progress-issue-N.md` 읽기

2. **업데이트 내용 결정**

   progress 파일에서 첫 번째 `⬜` 커밋 항목을 찾는다.
   git log의 최신 커밋이 해당 항목과 일치하면 업데이트 대상으로 확정.
   일치하지 않으면 사용자에게 어떤 커밋을 완료 처리할지 묻는다.

3. **파일 업데이트**

   해당 커밋 항목을:
   - `⬜` → `✅`
   - 커밋 해시 추가: `— \`<type>: ...\` (\`abc1234\`)`

   예시:

   ```
   ### ✅ Commit 1 — `chore: checkin-images Storage 버킷 마이그레이션 추가` (`29247ff`)
   ```

   이미 반영된 리뷰 피드백이 있으면 해당 항목에도 ✅ 표시.

4. **완료 안내**

   업데이트된 진행 현황을 요약해서 보여준다:
   - 완료된 커밋 수 / 전체 커밋 수
   - 다음 작업 커밋 제목
   - 남은 세션에서 계속 진행할지, 새 세션에서 재개할지 선택 안내:
     > 계속 진행할까요, 아니면 여기서 마무리하고 다음 세션에서 재개할까요?
